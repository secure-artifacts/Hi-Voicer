# Hi-Voicer 环境准备

这份说明用于换电脑时复现开发环境。普通用户只需要安装发布包，不需要安装 Node.js、Rust 或 Visual Studio Build Tools。

## 普通用户运行环境

1. 从 `https://github.com/cg202601/Hi-Voicer/releases` 下载 GitHub Actions 自动构建的发布包，优先使用 `Hi-Voicer_1.0.10_x64-setup.exe`，也可以使用同版本 MSI 包。
2. Windows 需要 Microsoft Edge WebView2 Runtime。多数 Windows 10/11 已经自带。
3. 首次配置模型时需要联网下载 Sherpa-ONNX CPU 运行时和模型文件；配置完成后，识别和转写在本地运行。
4. ffmpeg 不会自动下载。需要转码、字幕音频段导出、双轨混音或基础音频处理时，请先安装本机 ffmpeg，或把 `ffmpeg.exe` 放到应用数据目录/程序目录的 `engines\ffmpeg` 下。
5. The release package does not bundle CUDA runtime and will not auto-download GPU runtimes during transcription. CPU is the default stable path; DirectML must be evaluated per machine from Diagnostics before it is treated as verified.
6. 普通用户不需要配置 Python、Node.js、Rust、Visual Studio Build Tools 或环境变量。

## 开发需要安装的软件

1. Node.js 20 或更新版本。
2. Rust 1.96.0，Windows MSVC 工具链。
3. Microsoft Visual Studio 2022 Build Tools，包含 C++ Build Tools。
4. Microsoft Edge WebView2 Runtime。

## 新电脑快速步骤

```powershell
cd A:\Gmini\Hi-Voicer
.\scripts\setup-windows.ps1
```

如果系统禁止执行脚本，可以只对当前 PowerShell 窗口放开：

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\scripts\setup-windows.ps1
```
