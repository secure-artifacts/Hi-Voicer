# Hi-Voicer 环境准备

这份说明用于换电脑时复现开发环境。普通用户只需要安装发布包，不需要安装 Node.js、Rust 或 Visual Studio Build Tools。项目目录建议保持在 `A:\Gmini\Hi-Voicer` 这种独立目录里，Git、依赖、Tauri 配置都放在项目里面，避免和其它软件插件目录混在一起。

## 普通用户运行环境

1. 从 `https://github.com/cg202601/Hi-Voicer/releases` 下载 GitHub Actions 自动构建的发布包，优先使用 `Hi-Voicer_1.0.5_x64-setup.exe`，也可以使用同版本 MSI 包。
2. Windows 需要 Microsoft Edge WebView2 Runtime。多数 Windows 10/11 已经自带；如果目标电脑没有，需要额外安装。
3. 首次配置模型时需要联网下载 Sherpa-ONNX CPU 运行时和模型文件；配置完成后，识别和转录在本地运行。
4. ffmpeg 不会自动下载。需要转码、字幕音频段导出、双轨混音或基础音频处理时，请先安装本机 ffmpeg，或把 `ffmpeg.exe` 放到应用数据目录/程序目录的 `engines\ffmpeg` 下。
5. CUDA runtime 不会在转录过程中自动下载；CUDA 没准备好时会直接回退 CPU。
6. 普通用户不需要配置 Python、Node.js、Rust、Visual Studio Build Tools，也不需要手动配置环境变量。

## 开发需要安装的软件

1. Node.js 20 或更新版本。
2. Rust 1.96.0，Windows MSVC 工具链。
3. Microsoft Visual Studio 2022 Build Tools，包含 C++ Build Tools。
4. Microsoft Edge WebView2 Runtime。

Tauri v2 的 Windows 前置条件是 Microsoft C++ Build Tools、Microsoft Edge WebView2 和 Rust。当前项目已经把 npm 依赖锁在 `package-lock.json`，Rust 工具链锁在 `rust-toolchain.toml`。

## 新电脑快速步骤

在 PowerShell 里进入项目目录：

```powershell
cd A:\Gmini\Hi-Voicer
```

如果已经安装好 Node.js 和 Visual Studio Build Tools，可以直接运行：

```powershell
.\scripts\setup-windows.ps1
```

如果系统禁止执行脚本，可以只对当前 PowerShell 窗口放开：

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\scripts\setup-windows.ps1
```

## 检查环境

```powershell
.\scripts\check-env.ps1
```

应该看到这些项目通过：

```text
Node.js
npm
rustc
cargo
Visual Studio C++ Build Tools
Microsoft Edge WebView2 Runtime
```

## 常用验证命令

```powershell
npm ci
npm test
npm run build
cargo check --manifest-path src-tauri\Cargo.toml
npm run tauri -- --version
```

正式桌面包构建：

```powershell
npm run tauri -- build
```
